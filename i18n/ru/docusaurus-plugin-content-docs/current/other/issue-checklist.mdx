Прежде чем сообщать об ошибке, ознакомьтесь с данным руководством, чтобы попытаться самостоятельно ее устранить.

### Куда сообщать об ошибке

Для сообщения об ошибке используйте [трекер ошибок на GitHub](https://github.com/Floorp-Projects/Floorp/issues).

**Сообщества Floorp**

Если вы не уверены, является ли возникшая ситуация ошибкой, вы можете обратиться к сообществам Floorp:

* Reddit r/Floorp: https://www.reddit.com/r/Floorp/
* Discord Ablaze: https://aka.ablaze.one/discord

**Важно!** При обращении к сообществу Reddit **обязательно** прочитайте пост с названием [Do not post Issues or Feature Requests in Reddit.](https://www.reddit.com/r/Floorp/comments/19f7762/do_not_post_issues_or_feature_requests_in_reddit/?utm_source=share&utm_medium=web3x&utm_name=web3xcss&utm_term=1&utm_content=share_button). Сообщества предназначены для проверки актуальности возникшей проблемы, а не для официального сообщения об ошибках.

### Как проверить, является ли это ошибкой

#### 1. Проверка существующих ошибок

Прежде чем сообщать об ошибке, проверьте, не описана ли она уже кем-то другим. Дублирующие отчеты могут замедлить процесс решения проблемы.

**Проверка существующих ошибок возможна двумя способами:**

* Посетите официальный [трекер ошибок Floorp](/docs/category/issue-tracker)
* Воспользуйтесь поиском по ключевым словам на GitHub Issues.

#### 2. Проверка проблемы в базовой версии

Иногда проблема может быть связана с пользовательскими настройками или установленными расширениями.

**Выполните следующие действия, чтобы диагностировать проблему:**

- **Обновите Floorp до последней версии:** Проблема могла быть исправлена в последних обновлениях.

- **Отключите все расширения браузера**: Конфликт с расширениями может быть причиной ошибки. Если отключение расширений решило проблему, обратитесь к разработчику расширения или найдите альтернативное.

- **Очистите кэш и куки браузера:** Проблема может быть связана с устаревшими данными в кэше.

- **Создайте новый профиль браузера**

Профиль браузера — это отдельный набор настроек, данных и журналов. Проблема может быть связана с изменениями в структуре хранения данных при обновлении Floorp.

[Инструкции по созданию профиля в Firefox](https://support.mozilla.org/en-US/kb/profile-manager-create-remove-switch-firefox-profiles)

#### 3. Сравнение с оригинальным Firefox

Floorp основан на версии Firefox ESR (Long-Term Support Version).

**Проверьте, возникает ли проблема в последней версии Firefox ESR:**
* Если проблема наблюдается и в Firefox ESR, сообщите об ошибке в системе отслеживания ошибок Mozilla.

**Исключение**:
Данный пункт можно пропустить, если проблема касается функций, специфичных для Floorp.

#### 4. Сообщение об ошибке на GitHub

**Благодарим вас за то, что вы проверили возникшую проблему, следуя данному руководству!**

Для официального сообщения об ошибке перейдите на страницу [GitHub Issue](https://github.com/Floorp-Projects/Floorp/issues).

### Полезная информация для решения проблемы
- **Подробные пошаговые инструкции по воспроизведению ошибки**

  Опишите последовательность действий, приводящих к возникновению ошибки.

  ```text
  Действия:

  1. Нажал кнопку "Настройки"
  2. Перешёл в раздел "Внешний вид"
  3. Выбрал тему "Floorp Fluerial"

  Результат:
    * Сломалось оформление панели вкладок
  ```

- **Снимки экрана с демонстрацией проблемы**

- **Логи консоли браузера**

Откройте Devtools сочетанием клавиш
* На Windows: `Ctrl` + `Shift` + `I`
* На MacOS: `Cmd` + `Option` + `I`

Дополнительно откройте панель Browser Toolbox сочетанием

* На Windows: `Ctrl` + `Alt` + `Shift` + `I`
* На MacOS: `Cmd` + `Option` + `Shift` + `I`.

* Логи из Browser Toolbox могут быть полезны для диагностики.

[Документация Browser Toolbox](https://firefox-source-docs.mozilla.org/devtools-user/browser_toolbox/index.html)
